# Workflow 初学者入门 PLAN（6 周）

## 目标
- 从“会 C++ 语法”过渡到“能读懂并改动 Workflow 服务端示例”。
- 建立异步任务流思维：任务创建、启动、回调、错误处理、超时与资源回收。
- 最终独立完成一个小型 HTTP 聚合服务（可运行、可观测、可扩展）。

## 学习节奏
- 每周 5 天，每天 1~2 小时。
- 每天结构：20 分钟读文档 + 40 分钟改代码 + 20 分钟验证输出 + 10 分钟复盘。

## 第 1 阶段：项目总览与第一个异步任务（第 1 周）
### 你会获得
- 看懂项目模块地图。
- 跑通并改动 `tutorial-01-wget.cc`。
- 能解释 `get_state()/get_error()` 的基础错误语义。

### 学习路径
1. 阅读项目总览：`README_cn.md`
2. 阅读教程：`docs/tutorial-01-wget.md`
3. 对照源码：`tutorial/tutorial-01-wget.cc`
4. 错误机制：`docs/about-error.md`

### 实操作业
- 任务 A：修改 `User-Agent`，验证服务端收到新值。
- 任务 B：打印响应体长度与状态码。
- 任务 C：输入一个错误域名，记录 `state/error` 组合。

### 验收标准
- 可以说清楚：工厂创建任务不返回空；错误在 callback 暴露。
- 可以通过输出区分：成功、DNS 错误、系统错误。

## 第 2 阶段：HTTP 服务端入门（第 2 周）
### 你会获得
- 会写 `process(WFHttpTask*)` 处理函数。
- 理解一个请求在连接上的序号 `get_task_seq()`。

### 学习路径
1. 阅读教程：`docs/tutorial-04-http_echo_server.md`
2. 对照源码：`tutorial/tutorial-04-http_echo_server.cc`

### 实操作业
- 任务 A：新增 `/health` 路由返回 `ok`。
- 任务 B：为 `/echo` 返回请求头。
- 任务 C：在日志中打印客户端 IP、端口、请求路径。

### 验收标准
- 使用 `curl` 可稳定访问两个路径并得到不同响应。
- 明确“server 的 process 与 client 的 callback 都处理 WFHttpTask，但语义不同”。

## 第 3 阶段：并发与任务流（第 3~4 周）
### 你会获得
- 掌握并行任务、DAG 任务、超时控制。

### 学习路径
1. `docs/tutorial-06-parallel_wget.md`
2. `docs/tutorial-11-graph_task.md`
3. `docs/about-timeout.md`
4. `docs/about-exit.md`

### 实操作业
- 任务 A：并发请求 3 个 URL，统计最快/最慢响应。
- 任务 B：构建一个小 DAG：先并发下载，再合并结果。
- 任务 C：给任务设置超时并分类超时原因。

### 验收标准
- 你能画出任务依赖图，并和代码一一对应。

## 第 4 阶段：服务端小项目（第 5~6 周）
### 你会获得
- 完成一个最小可用异步服务端项目。

### 项目题目
- mini-aggregator：
  - 入参：`/aggregate?q=...`
  - 行为：并发访问两个上游（可先用两个公开 HTTP 接口模拟）
  - 出参：合并后的 JSON
  - 稳定性：超时、错误分类、日志

### 验收标准
- 能跑压测（轻量），并输出成功率和平均延迟。
